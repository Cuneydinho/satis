İdefix API Portal' e Hoşgeldiniz
Nasıl Entegrasyon Yapılır?
API portalı entegrasyon/API servisleri üzerinden çalışacak Merchant/Entegratörler için oluşturulan; ürün aktarımı, stok ve fiyat güncellemesi, sipariş işlemleri, fatura gönderimi ,iade işlemleri gibi bir çok işlemin yapılmasına olanak sağlayan servislerin dokümanıdır.

Servis Limitleri
API servislerinde eşiğe ulaşıldığında request 429 Too Many Request statüsü ile sonuçlanır ve aşağıdaki response header'ları dikkate alarak istek sıklığınızı düzenlemelisiniz.

➜ X-Api-Call-Limit → saniyedeki istek sayısını gösterir.
➜ Retry-After: 2.0 → buradaki değer kadar saniye sonra tekrar deneyin.

Canlı - Test Ortam Bilgileri

Auth için kullanılan API KEY, API SECRET KEY ve Satıcı ID bilgilerinize Satıcı panelinde yer alan "Hesap Bilgilerim" altında "Entegrasyon Bilgileri" sayfasından "Yeni API Oluştur" butonuna tıklayarak ulaşabilirsiniz. Bu butona tıklandığında API KEY ve API SECRET KEY bilgileri satıcının üzerinde kayıtlı olan mail adresine gönderilecektir. Bu bilgiler prod ve stage ortamda değişmektedir.

Idefix test ortamına erişim için IP yetkilendirmesi gerekmektedir. Prod ortamında herhangi bir IP yetkilendirmesine gerek yoktur. Hem test hem de prod ortamında herhangi bir erişim sorunuyla karşılaşırsanız size iletilen destek maili üzerinden iletişime geçebilirsiniz.

TEST ORTAM BİLGİLERİ
PIM PATTERN

https://ide-pimapi.idefiks.net/api/connector/product-category
OMS PATTERN

https://ide-omsapi.idefiks.net/api/shipment/connect/{vendorID}/claim-list
1. Adım: idefix test ortamına erişim için IP yetkilendirmesi gerekmektedir. Test ortamına erişebilmek için IP bilgileri idefix tarafına iletilerek yetki tanımı yapılmalıdır.

2. Adım: Test ortamının kullanılabilir olması için IP tanımı ile birlikte, test hesaplarının idefix tarafından tanımlanıp sizlerle paylaşılması gerekmektedir.


3. Adım: Test hesapları oluşturulduktan sonra, API bilgilerine satıcı panelinde yer alan “Hesap Bilgilerim” menüsü altındaki “Entegrasyon Bilgileri” sayfasından erişebilirsiniz.

❗ NOT: Testlerin öncelikle stage ortamda tamamlanıp, ardından canlı (prod) ortamda yapılmasını rica ederiz.
CANLI ORTAM BİLGİLERİ
PIM PATTERN

https://merchantapi.idefix.com/pim/product-category
OMS PATTERN

https://merchantapi.idefix.com/oms/{vendorId}list
❗ Ortam Uyarısı: API KEY, API SECRET KEY ve Satıcı ID bilgileri prod ve stage ortamları için farklıdır.


Sıkça Sorulan Sorular

▶
Stok/fiyat gönderdim ama idefix sistemine yansımadı, ne yapmam gerekiyor?
Öncelikle stok-ve-fiyat-guncel-durum-sorgulama servisinden bir hata alınıp alınmadığı kontrol edilmelidir. Eğer bir hata yok ise stok/fiyat kontrolünün doğru ekrandan yapıldığına emin olunur. (Mcenter→Envanterim) ya da inventory-list servisinden kontroller yapılabilir.
▶
Ürün bilgilerine ait güncelleme işlemlerini nasıl yapabilirim ?
API servisleri üzerinden ürüne ait yalnızca stok-fiyat güncellemeleri yapılabilmektedir. Ürüne ait diğer bilgiler API aracılığıyla güncellenmemektedir.
▶
İdefix sisteminde bulunan güncel stok/fiyat bilgilerime hangi servisten ulaşabilirim?
inventory-list servisinden envantere eklenmiş ürünlerinizin güncel stok/fiyat bilgilerine erişebilirsiniz.
▶
Fatura kesilebilmesi için bireysel/kurumsal fatura bilgisi ve indirim tutar bilgilerine hangi servisten ulaşabilirim?
Sevkiyatlar Listesi (list) servisinden bu bilgi dönmektedir. Kurumsal fatura için "isCommercial" değeri kontrol edilmelidir.

Kurumsal: company, taxNumber, taxOffice bilgileri invoiceAddress alanından alınabilir.

Bireysel: platformun uygulamış olduğu indirim ve kuponlar için response'ta ilgili parametreler 0 olarak döner.
▶
Ürünü servislerinizden oluşturduk fakat idefix sistemine yansımadı, ne yapmam gerekiyor?
Ürün idefix sisteminde yer almıyor ise create servisini tetiklediğinizden emin olun. Ardından batch-result servisinden hata kontrol edilmelidir. Gerekli düzenlemeler sonrası tekrar gönderim yapılabilir.
▶
Satıcı entegrasyonlu kargo modelinde çalışıyorum. Kargomu nasıl teslim edildi olarak bildirebilirim?
Teslim edildi bilgisi için manuel-deliver servisi kullanılır.
▶
Servislerinize bağlanmak için token nasıl alabilirim?
authorization sayfasından token alma ile ilgili bilgiye erişebilirsiniz.
▶
Bir sipariş içerisindeki ürün sayısını alabileceğimiz bir parametreniz var mı?
shipment-list servisinde her ürün max. 1 adet olarak döner. Quantity bilgisi yoktur. Ancak toplam ürün adetleri için https://merchantapi.idefix.com/oms/{vendorId}/list-v2 servisi kullanılabilir.
▶
Siparişime ait kargo profil bilgilerine nereden ulaşabilirim?
shipment-list servisinden cargoProfileId alanına erişilir. Detay için vendor/profile/list servisi kullanılabilir.
▶
Güncel sipariş statüleri nelerdir?
shipment-list servisinden güncel statülere erişebilirsiniz.
▶
Servislerinizdeki rate limitler nelerdir?
Eşik aşıldığında 429 Too Many Request hatası alınır. Aşağıdaki response header'lar dikkate alınmalıdır:

X-Api-Call-Limit: saniyedeki istek sayısını gösterir.
Retry-After: 2.0 → bu kadar saniye sonra tekrar deneyin.
▶
Bir ürün oluşturma isteği içerisinde kaç ürün olabilir, bir batch içerisinden max. kaç item döner?
Max. 200 item gönderilebilir. Bir batch içerisinde en fazla 200 item olur.
▶
Sipariş faturalarımı sisteme nasıl yükleyebilirim ?
1) Entegrasyon ile iletilmek istendiği durumda Fatura Linki Gönderme (invoice-link) servisi üzerinden shipmentID bilgisi kullanılarak yükleme yapılabilmektedir.

2) Manuel yükleme yapılmak istendiği durumda; MercentCenter -> Siparişlerim -> Fatura Yükle butonuyla işlem yapılabilmektedir.

3) Shipment-list servisinde bulunan maskeli mail adresine pdf/link gönderilerek işlem sağlanabilmektedir.
▶
Ürün statülerine nasıl erişebilirim ?
Ürün statülerine batch-result servisinden ulaşabilirsiniz.

Kategorilerin Alınması

Kategori Listesi (product-category)
create servisine yapılacak isteklerde gönderilecek kategori bilgilerine ait ID değerleri bu servis kullanılarak alınacaktır.

GET product-category
https://merchantapi.idefix.com/pim/product-category
Örnek Servis Cevabı:

{
[  
  {
    "id": 522,
    "parentId": 0,
    "subs": [
      {
        "id": 1192,
        "parentId": 522,
        "subs": [
          {
            "id": 545,
            "parentId": 1192,
            "subs": [],
            "name": "Salopet",
            "topCategory": 522
          },
          {
            "id": 546,
            "parentId": 1192,
            "subs": [],
            "name": "Tulum",
            "topCategory": 522
          }
        ], 
        "name": "Salopet & Tulum",
        "topCategory": 522
      }
    ]
  "name": "Salopet & Tulum",
  "topCategory": 522
  }
]
}
Response Parametreleri
Parametre Ad	Açıklama
id	Kategorinin idefix sistemindeki ID bilgisidir.
name	Kategorinin adıdır.
parentId	Kategorinin bağlı olduğu bir üst kategori bilgisidir.
subs	Bulunulan kategorinin alt kategorileridir. Eğer alt kategori yoksa subs alanı boş döner.
topCategory	Kategorinin bağlı olduğu 1. seviye kategori bilgisidir.


Kategori Özellik Listesi

Kategori Özellik Listesi (category-attribute)
create servisine yapılacak isteklerde gönderilecek attributes bilgileri ve detayları bu servis kullanılarak alınmalıdır.

GET category-attribute
https://merchantapi.idefix.com/pim/category-attribute/{categoryID}
Ürün create etmek için kategori özelliklerini almanız gerekecektir. Bunun için ise en alt seviyedeki kategori ID bilgisi kullanılmalıdır. Seçtiğiniz kategorinin alt kategorileri var ise bu kategori bilgisi ile ürün oluşturamazsınız.
Ürün create etmek için attributeleri çektiğinizde required = true olan alanlar ürün oluşturma isteğinde mutlaka gönderilmelidir. Aksi halde ürününüz idefix sisteminde oluşmayacaktır.
Örnek Servis Cevabı:
{
  "id": 546,
  "name": "Tulum",
  "categoryAttributes": [
    {
      "attributeId": 24,
      "attributeTitle": "Dahili Hafıza",
      "allowCustom": false,
      "categoryId": 546,
      "required": true,
      "isVariant": true,
      "isSlicer": true,
      "attributeValues": [
        {
          "id": 93301,
          "name": "1 TB"
        },
        {
          "id": 93300,
          "name": "6 GB"
        },
        {
          "id": 93299,
          "name": "1 GB"
        }
      ]
    }
  ]
}
Response Parametreleri
Parametre Ad	Açıklama
id	Kategorinin idefix sistemindeki ID bilgisidir.
name	Kategorinin idefix sistemindeki isim bilgisidir.
attributeId	Kategorideki özellik ID bilgisidir.
attributeTitle	Kategorideki özellik adıdır.
allowCustom	true ise ürün create edilirken ID yerine freetext (string) gönderim yapılmalıdır.
required	true ise ürün create edilirken ilgili özelliklerin attributes / attributeValues alanında gönderilmesi gerekmektedir.
isSlicer	true ise idefix.com önyüzünde ürünler bu özelliğe göre farklı şekilde listelenir.
isVariant	İlgili özelliğin varyant olup olmadığı bilgisidir. idefix.com’da ürün görünürken bu özelliğin varyant bilgisini ifade eder.
attributeValues.id	Kategorideki özellik değeri için girilebilecek değer ID bilgisidir.
attributeValues.name	Kategorideki özellik değeri için girilebilecek değer adıdır.

Hızlı Ürün Ekleme

Hızlı Ürün Ekleme(fast-listing)
Bu method idefix katalogunda kayıtlı olan ve global barkod içeren ürünlerinizi mapping sürecine gerek olmadan hızlı bir şekilde satışa açmanızı sağlar. İstek içerisinde gönderdiğiniz ürünler idefix kataloğunda bulunmuyor ise bu method ile ürünleriniz sisteme yüklenmez. Bu sebeple ilk defa açılacak ürünler için “Ürün Oluşturma” methodu kullanılmalıdır.

ℹ️ Bilgilendirme
1. idefix kataloğuna eklenmek istenen ürünler için, request içerisinde beklenen tüm bilgiler eksiksiz olarak gönderilmelidir.
2. Hızlı ürün yükleme işlemi sonrasında batchRequestId bilgisi döner. Bu ID ile hangi ürünlerin eşleştiği ve hangilerinin katalogda bulunmadığı için failed olduğu, Toplu hızlı ürün durumu sorgulama servisi üzerinden kontrol edilebilir.
3. Eşleşen ürünler satıcı paneli üzerinden kontrol edilerek onaylanabilir veya reddedilebilir. Aynı işlemler API üzerinden approve-item ya da declined-item servisleri ile de gerçekleştirilebilir.
vendor her satıcının unique bir tanımlayıcısıdır. Zorunlu bir alandır.
POST fast-listing
https://merchantapi.idefix.com/pim/catalog/{vendorId}/fast-listing
olarak belirtilenler zorunlu parametrelerdir.
Parametre Ad	Açıklama
barcode *	Ürün barkodu
title *	Ürün ismi
vendorStockCode *	Ürünün satıcı tarafındaki stok kodudur
price *	Ürünün idefix satış fiyatıdır
comparePrice	Ürünün tavsiye edilen satış fiyatıdır
inventoryQuantity *	Ürünün idefix satılabilir stok adedidir

Ürün aktarım işlemi sonrasında response içerisinde yer alan batchRequestId ile ürünlerinizin ve aktarım işleminin durumunu batch-result servisi üzerinden kontrol etmelisiniz.

Ürün Merchant Onay

Ürün Merchant Onayı(approve-item)
create servisine yapılan isteklerde idefix kataloğunda gönderdiğiniz ürün ile eşleşen statüsünde olan ürünlerin doğruluğunu onaylayıp ürün onaylama yapabileceğiniz servistir. Dilerseniz bu işlemi servis ile yapmadan merchant center backoffice ekranları üzerinden de yapabilirsiniz.

• Gönderilen ürün idefix sisteminde mevcut ise waiting_vendor_approve statüsüne düşer. Mevcut ürün bilgileri size sunulur ve gönderdiğiniz ürünler ile kontrolünüzün ardından onay veya red vermeniz beklenir.

• Gönderilen ürün idefix sisteminde mevcut değil ise not_matched statüsüne düşer. Bu durumda idefix ekiplerinin aksiyon alması beklenir. Göndermiş olduğunuz ürünlerin güncel durumunu öğrenmek için size dönen batchId ile batch-result servisini çağırmalısınız.

vendor her satıcının unique bir tanımlayıcısıdır. Zorunlu bir alandır.
POST approve-item
https://merchantapi.idefix.com/pim/pool/{vendorId}/approve-item

      "barcode": "string"

• Gönderilen ürün idefix sisteminde mevcut değil ise not_matched statüsüne düşer. Bu durumda idefix ekiplerinin aksiyon alması beklenir. Göndermiş olduğunuz ürünlerin güncel durumunu öğrenmek için size dönen batchId ile batch-result servisini çağırmalısınız.

Sipariş sevkiyatlarının alınması

shipment list/Sevkiyatlar Listesi (list)
idefix sistemine ilettiğiniz ürünler için müşteriler tarafından sipariş edilen ve ödeme kontrolünden geçmiş olan her shipmentin bilgisini bu method ile alabilirsiniz. Satıcı bazında oluşturulan siparişler dokümanlarda shipment olarak adlandırılacaktır.

Shipment datası alınırken gelen orderNumber değeri, ilgili shipment’ın idefix sistemindeki ana sipariş numarasına karşılık gelir. Aynı seviyede yer alan id değeri ise bu sipariş numarasına bağlı oluşturulmuş sevkiyatı temsil eder.
vendor her satıcının unique bir tanımlayıcısıdır. Zorunlu bir alandır.
Bilgilendirme: Kurumsal fatura için isCommercial değeri kontrol edilmelidir. Kurumsal fatura kesilmesi gereken durumda bu değer "1" olarak döner, bireysel fatura olması durumunda " " (boş) olarak döner. (isCommercial = 1) durumunda "company", "taxNumber" ve "taxOffice" bilgilerini invoiceAddress alanından alabilirsiniz. Bireysel faturada (isCommercial = " ") platformun uyguladığı indirim ve kuponlar response içerisinde ilgili parametrelerde 0 olarak döner. isLikeCommercial parametresi bireysel müşterinin kurumsal fatura talep ettiği durumlarda döner ve isCommercial parametresi ile aynı mantıkta çalışır.
GET list
https://merchantapi.idefix.com/oms/{vendorId}/list
* ile belirtilen alanlar zorunlu parametrelerdir.

Parametre Ad	Açıklama
ids	Shipment numarası bazında sorgulama yapılabilir. Birden fazla değer virgül (,) ile gönderilebilir.
orderNumber	Shipment’ın idefix sistemindeki ana sipariş numarasına karşılık gelir.
state	Shipment statüsüne göre sorgulama yapılabilir. Gönderilebilecek state değerleri response tablosunda yer almaktadır.
startDate	Belirli bir tarihten sonraki shipment’lar için sorgulama yapılabilir. Örnek format: 2022/09/30 23:59:59
endDate	Belirtilen tarihe kadar olan shipment’lar için sorgulama yapılabilir. Örnek format: 2022/09/30 23:59:59
lastUpdatedAt	Shipment’ın durumunun son güncellenme tarihini baz alır. Örnek format: 2022/09/30 23:59:59
page	Sadece belirtilen sayfadaki verileri döndürür. Default değer: 1
limit	Bir sayfada listelenecek maksimum kayıt adedini belirtir. Default değer: 10
sortByField	Sıralama alanını belirtir. Varsayılan olarak shipment ID’ye göre sıralanır. id, createAt, updateAt parametreleri ile sorgulanabilir.
sortDirection	Sıralama yönünü belirtir. Varsayılan olarak güncel ID’den eski ID’ye doğru sıralanır. desc, asc parametreleri ile sorgulanabilir.
vendor *	İlgili satıcının ID bilgisidir.

Tedarik Edilemedi Bildirimi

Tedarik Edilemedi Bildirimi(unsupplied)

Tedarik edilemedi bilgisi iletildikten sonra ilgili shipment içerisinde ürün kalması durumunda idefix sistemi tarafından aynı orderNumber üzerinde yeni bir shipmentId oluşur. POST unsupplied
https://merchantapi.idefix.com/oms/{vendorId}/{shipmentId}/unsupplied
* ile belirtilen alanlar zorunlu alanlardır.

Parametre Ad	Açıklama
shipmentID*	Tedarik edilemeyen ilgili shipment ID bilgisidir.
id*	Shipment içerisindeki tedarik edilemeyen item ID bilgisidir.
reasonId*	Tedarik edilemeyen shipment’in sebep ID bilgisidir. Bu sebeplere reasons/noship servisinden erişebilirsiniz.

{
